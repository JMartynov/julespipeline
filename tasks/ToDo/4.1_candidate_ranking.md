# Task: 4.1 Candidate Ranking

### Objective
Create a mathematical evaluation utility that analyzes all executed candidates against the Baseline Score and ranks them based on performance improvement.

### Utility Design & Modularity
*   **Module Name:** `src/reporting/ranker.py`
*   **Independence:** Takes a Run Context/list of Metric Dictionaries. Outputs a sorted list of candidates with computed delta percentages.
*   **Configuration:** Scoring weights (e.g., penalize `COLLSCAN` vs `executionTimeMillis`) in `config/reporting.yaml`.

### Implementation Details
*   Calculate percentage change for `executionTimeMillis` and `totalDocsExamined`.
*   Apply configured penalty weights to determine the absolute "Winner".

### Testing Strategy
1.  **Unit Tests:** Provide hardcoded mock stats. Assert sorting correctly applies weights.
2.  **Comprehensive Acceptance Tests (BDD):** `tests/features/candidate_ranking.feature`
    *   *Scenario 1 (Happy Path):* Given 3 candidates with varying latency, when ranked, then the candidate with the lowest latency and zero in-memory sorts is ranked #1.
    *   *Scenario 2 (Edge Case - Tie Breaker):* Given Candidate A and B have identical latency, when ranked, then the algorithm falls back to `totalDocsExamined` or `totalKeysExamined` to break the tie.
    *   *Scenario 3 (Edge Case - Worse than Baseline):* Given all LLM candidates performed *worse* than the baseline query, when ranked, then the Baseline is explicitly flagged as the winner (indicating no optimization is recommended).
    *   *Scenario 4 (Edge Case - Memory Sort Penalty):* Given Candidate A is 5ms faster but uses an in-memory sort (`hasSortStage: true`), and Candidate B is 5ms slower but is a fully covered query, when ranked, then Candidate B wins due to the YAML penalty weights.

### Acceptance Criteria
* [ ] Utility is completely modular and can be instantiated independently.
* [ ] No hardcoded configuration parameters exist in the source code.
* [ ] Unit tests cover 90%+ of the logic.
* [ ] All BDD scenarios in the `.feature` file are written and passing.
