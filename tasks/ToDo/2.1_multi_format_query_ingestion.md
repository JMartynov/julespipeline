# Task: 2.1 Multi-Format Query Ingestion

### Objective
Build a modular query parser that accepts user-provided slow queries in either strict JSON format or native `mongosh` syntax, securely translating them into internal PyMongo kwargs.

### Utility Design & Modularity
*   **Module Name:** `src/benchmarking/query_parser.py`
*   **Independence:** Takes a string (JSON or JS) and outputs a safe, normalized dictionary representing the query (e.g., `{"filter": {...}, "sort": {...}, "limit": ...}`).
*   **Configuration:** Supported query operators and regex parsing patterns should be configured/loaded via `config/system.yaml`.

### Implementation Details
*   **JSON Path:** Directly parse standard `.json` files.
*   **Mongosh Path:** Use regex or a lightweight AST parser to translate strings like `db.collection.find({status: "A"}).sort({date: -1})` into `{"filter": {"status": "A"}, "sort": {"date": -1}}`.
*   Crucially, this module must NOT use `eval()` or `exec()` for security reasons.

### Testing Strategy
1.  **Unit Tests:** Provide dozens of edge-case strings (both JSON and Mongosh) and assert the output dictionaries perfectly match PyMongo execution requirements.
2.  **Comprehensive Acceptance Tests (BDD):** `tests/features/query_ingestion.feature`
    *   *Scenario 1 (Happy Path - JSON):* Given a valid JSON query file, when ingested, then it is perfectly mapped to PyMongo kwargs.
    *   *Scenario 2 (Happy Path - Mongosh):* Given a raw string `db.users.find({age: {$gt: 18}}).limit(5)`, when ingested, then it is parsed into `{"filter": {"age": {"$gt": 18}}, "limit": 5}`.
    *   *Scenario 3 (Edge Case - Aggregation):* Given a `db.collection.aggregate([...])` array, when ingested, then it recognizes the pipeline format and routes it to PyMongo's `aggregate` method instead of `find`.
    *   *Scenario 4 (Security - Reject Destructive Ops):* Given a query like `db.collection.drop()`, when ingested, then an `UnsafeQueryError` is immediately raised.
    *   *Scenario 5 (Security - Reject Eval):* Given an attempted Python injection `{"$where": "os.system('rm -rf /')" }`, when ingested, the parser sanitizes or rejects execution.

### Acceptance Criteria
* [ ] Utility is completely modular and can be instantiated independently.
* [ ] No hardcoded configuration parameters exist in the source code.
* [ ] Unit tests cover 90%+ of the logic.
* [ ] All BDD scenarios in the `.feature` file are written and passing.
