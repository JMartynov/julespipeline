# Task: 1.4 Generator Distribution Update

### Objective
Update the existing Data Generators (from Module 2) to consume the statistical profile from Task 1.2, ensuring synthetic data mathematically mirrors the live production data.

### Utility Design & Modularity
*   **Module Name:** `src/task2/generators/distribution_injector.py` (or modify existing generator class)
*   **Independence:** Takes a JSON Schema and a Statistical Profile Dictionary, and outputs populated Pydantic models/JSON instances.
*   **Configuration:** Random seed control for deterministic generation should be configurable in `config/system.yaml`.

### Implementation Details
*   Modify the data generation loop. If a field exists in the Statistical Profile, use `numpy.random.choice` (or Python's `random.choices`) to select values based on the profiled weights (probabilities) rather than purely uniform random selection.

### Testing Strategy
1.  **Unit Tests:** Feed a static profile (`{"A": 0.8, "B": 0.2}`) and verify that calling the generator 1000 times produces roughly 800 'A's and 200 'B's.
2.  **Comprehensive Acceptance Tests (BDD):** `tests/features/generator_distribution.feature`
    *   *Scenario 1 (Happy Path):* Given a schema and a 70/30 probability profile, when 10,000 records are generated, then the resulting dataset matches the 70/30 distribution within a 2% margin of error.
    *   *Scenario 2 (Edge Case - Missing Profile):* Given a schema without a provided profile, when generated, then it gracefully falls back to the legacy uniform random generation.
    *   *Scenario 3 (Edge Case - Impossible Constraints):* Given a schema requiring uniqueness (e.g., UUID) but a profile demanding duplicate frequencies, when generated, then the uniqueness constraint overrides the frequency distribution.

### Acceptance Criteria
* [ ] Utility is completely modular and can be instantiated independently.
* [ ] No hardcoded configuration parameters exist in the source code.
* [ ] Unit tests cover 90%+ of the logic.
* [ ] All BDD scenarios in the `.feature` file are written and passing.
